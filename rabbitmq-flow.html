正常消息链路

📤 Producer
发送扣库存消息 ──stock.deducted──► 🔀 stock.exchange
DirectExchange ──stock.deducted──► 📬 stock.deducted.queue ────────────► ⚙️ Consumer
扣减库存

生产者用 Routing Key "stock.deducted" 发消息 → Exchange 匹配 Binding Key → 进入主队列 → 消费者处理 → ACK 确认

消费失败 → 死信链路

📬 stock.deducted.queue
重试耗尽 ──自动转发──► 🔀 stock.dead.exchange
DirectExchange ──stock.dead──► 💀 stock.dead.queue ────────────► 🔧 人工排查

消费失败后由 x-dead-letter-exchange 参数自动路由，消息不丢失，等待人工处理

关键配置说明

Queue 声明（QueueBuilder）
.withArgument("x-dead-letter-exchange", DEAD_EXCHANGE)
.withArgument("x-dead-letter-routing-key", DEAD_ROUTING_KEY)
告诉 RabbitMQ：消息死亡后转到哪个 Exchange，用什么 Routing Key
Binding 声明（BindingBuilder）
BindingBuilder
 .bind(stockDeductedQueue)
 .to(stockExchange)
 .with(STOCK_DEDUCTED_KEY)
这里的 .with() 参数就是 Binding Key，Exchange 用它和 Routing Key 做匹配
